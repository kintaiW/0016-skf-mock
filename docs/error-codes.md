# SKF Mock 服务错误码文档

> 标准依据：GM/T 0016—2012 《智能密码钥匙密码应用接口规范》

---

## 1. 返回码格式说明

SKF 接口的所有函数均返回一个 `ULONG` 类型的返回值：

| 返回值 | 含义 |
|--------|------|
| `0x00000000` | 操作成功 |
| 非零值 | 操作失败，具体错误见下表 |

错误码按高位字节分类：

| 高位字节段 | 分类 |
|-----------|------|
| `0x01xxxxxx` | 设备/应用/容器相关错误 |
| `0x04xxxxxx` | 算法/数据处理错误 |
| `0x05xxxxxx` | 文件操作错误 |
| `0x0Axxxxxx` | PIN 认证相关错误 |
| `0x0FFFFFFF` | 通用失败 |

## 2. 错误码速查表

### 2.1 通用错误

| 错误码（十六进制） | 名称 | 含义 | 可能原因 | 解决方案 |
|-------------------|------|------|----------|----------|
| `0x00000000` | `SAR_OK` | 操作成功 | — | — |
| `0x0FFFFFFF` | `SAR_FAIL` | 通用失败 | 内部逻辑错误、非预期状态、底层调用失败 | 查看 Mock 服务日志，确认调用参数和调用顺序是否正确 |
| `0x01000002` | `SAR_MEMORYERR` | 内存错误 | 申请内存失败、内存不足、指针为 NULL | 检查传入的缓冲区指针是否为空；确认系统剩余内存是否充足 |

### 2.2 设备与会话错误

| 错误码（十六进制） | 名称 | 含义 | 可能原因 | 解决方案 |
|-------------------|------|------|----------|----------|
| `0x01000001` | `SAR_NODEVICE` | 设备不存在 | 未找到可用的密码钥匙设备；Mock 服务未启动；设备已被拔出 | 确认 Mock 服务正在运行（`cargo run` 或对应端口监听中）；检查设备枚举接口 `SKF_EnumDev` 返回结果 |
| `0x01000004` | `SAR_NOAPPLICATION` | 应用不存在 | 指定名称的应用尚未创建；应用名拼写错误 | 先调用 `SKF_CreateApplication` 创建应用；使用 `SKF_EnumApplication` 枚举已有应用名称 |
| `0x01000005` | `SAR_NOCONTAINER` | 容器不存在 | 指定名称的容器尚未创建；容器已被删除 | 先调用 `SKF_CreateContainer` 创建容器；使用 `SKF_EnumContainer` 确认容器存在 |
| `0x01000008` | `SAR_KEYNOTFOUND` | 密钥未找到 | 容器内无对应类型密钥（加密密钥/签名密钥）；密钥尚未导入或生成 | 确认已调用 `SKF_GenECCKeyPair` 或 `SKF_ImportECCKeyPair` 生成或导入密钥 |

### 2.3 PIN 认证错误

| 错误码（十六进制） | 名称 | 含义 | 可能原因 | 解决方案 |
|-------------------|------|------|----------|----------|
| `0x0A000003` | `SAR_INVALIDPIN` | PIN 错误 | 用户输入的 PIN 值与设备存储值不匹配 | 确认使用正确的 PIN；Mock 服务默认管理员 PIN 通常为 `12345678`，用户 PIN 为 `123456`，请查阅 `README.md` 中的默认配置 |
| `0x0A000004` | `SAR_USERUNLOCKED` | 用户已解锁（无需再次验证） | 当前会话已完成 PIN 验证，重复调用验证接口 | 确认会话状态，无需重复调用 `SKF_VerifyPIN` |
| `0x0A000005` | `SAR_PIN_LOCKED` | PIN 已锁定 | 连续多次输入错误 PIN，超过最大重试次数 | 使用管理员 PIN 调用 `SKF_UnblockPIN` 解锁；Mock 环境下可重启服务重置状态 |

### 2.4 算法与数据处理错误

| 错误码（十六进制） | 名称 | 含义 | 可能原因 | 解决方案 |
|-------------------|------|------|----------|----------|
| `0x04000001` | `SAR_ALGOIDNOTSUPPORT` | 算法不支持 | 传入的算法标识符不在 Mock 服务支持列表中 | 查阅 Mock 服务支持的算法列表（SM1、SM2、SM3、SM4）；确认 `AlgId` 参数值正确 |
| `0x04000003` | `SAR_PADERROR` | 填充错误 | 数据填充方式不符合预期；密文长度与块大小不对齐 | 检查加密前的数据填充逻辑；SM4 分组加密需保证数据为 16 字节对齐 |

### 2.5 文件操作错误

| 错误码（十六��制） | 名称 | 含义 | 可能原因 | 解决方案 |
|-------------------|------|------|----------|----------|
| `0x05000001` | `SAR_OPENFILEERR` | 文件打开失败 | 试图读写容器内文件时失败；文件不存在或权限不足 | 确认文件名正确；确认应用/容器已正确打开；Mock 服务以足够权限运行 |

## 3. 调试方法

### 3.1 查看 Mock 服务日志

SKF Mock 服务使用 Rust 标准日志框架，通过环境变量控制日志级别：

```bash
# 以 debug 级别启动，输出所有接口调用详情
RUST_LOG=debug cargo run

# 仅查看 warn 及以上级别（生产建议）
RUST_LOG=warn cargo run
```

日志中每次接口调用均会打印：
- 调用的函数名（如 `SKF_VerifyPIN`）
- 传入的关键参数值
- 返回的错误码（十六进制）

### 3.2 在集成测试中捕获错误码

```c
ULONG rv = SKF_VerifyPIN(hApplication, USER_TYPE, "123456", &retryCount);
if (rv != SAR_OK) {
    fprintf(stderr, "SKF_VerifyPIN failed: 0x%08X\n", rv);
    // 对照上方错误码表进行排查
}
```

### 3.3 Mock 服务状态重置

Mock 服务为内存状态，重启即可清空所有应用、容器、密钥：

```bash
# 停止服务
Ctrl+C

# 重新启动（状态归零）
cargo run
```

## 4. 常见问题排查流程

### 4.1 调用返回 `SAR_NODEVICE (0x01000001)`

```
1. 确认 SKF Mock 服务已启动
   └─ 检查进程：ps aux | grep skf-mock
   └─ 检查端口：ss -tlnp | grep <端口号>

2. 确认客户端连接地址与 Mock 服务监听地址一致

3. 确认调用了 SKF_EnumDev 并成功获取设备名
```

### 4.2 调用返回 `SAR_INVALIDPIN (0x0A000003)`

```
1. 确认当前使用的 PIN 类型（管理员 PIN / 用户 PIN）
   └─ 管理员 PIN：用于管理操作，如创建/删除应用
   └─ 用户 PIN：用于签名、加解密等业务操作

2. 查看 Mock 服务日志，确认收到的 PIN 值

3. 对照 README.md 中的默认 PIN 配置
```

### 4.3 签名或加密操作返回 `SAR_KEYNOTFOUND (0x01000008)`

```
完整调用顺序（缺任何一步均可能导致此错误）：

SKF_EnumDev          → 枚举设备
SKF_ConnectDev       → 连接设备
SKF_OpenApplication  → 打开应用（或先创建）
SKF_VerifyPIN        → 验证用户 PIN
SKF_OpenContainer    → 打开容器（或先创建）
SKF_GenECCKeyPair    → 生成密钥对（SM2）
                       ↑ 缺少此步骤将导致 SAR_KEYNOTFOUND
SKF_ECCSignData      → 签名
```

### 4.4 调用返回 `SAR_PIN_LOCKED (0x0A000005)`

```
1. Mock 服务重启即可解锁（内存状态）

2. 若需在不重启的情况下解锁：
   └─ 调用 SKF_UnblockPIN，传入管理员 PIN 和新用户 PIN

3. 检查测试代码是否存在循环重试错误 PIN 的逻辑
```

### 4.5 通用排查步骤

```
发现非零返回码
      │
      ├─ 对照本文档错误码表，确认错误类别
      │
      ├─ 开启 RUST_LOG=debug，重现问题
      │
      ├─ 检查调用序列是否完整（设备→应用→容器→PIN→操作）
      │
      ├─ 检查参数：缓冲区长度、指针是否为 NULL、算法 ID
      │
      └─ 仍无法解决：提交 Issue，附上日志和调用代码片段
```

## 5. 错误码与 GM/T 0016 标准对照

本 Mock 服务的错误码与 GM/T 0016—2012 附录 A 中定义的错误码完全一致，便于与真实设备进行对比测试。如需查阅完整的标准错误码列表，请参考规范原文附录部分。
